# Contributing to Incr

Thank you for your interest in contributing to Incr.

## How to contribute

The easiest ways to contribute are:

* open a GitHub issue to report a bug, documentation problem, or reproduction issue
* open a pull request with a fix or improvement
* share artifact-evaluation feedback, especially around setup, portability, and reproducibility

## Development expectations

When contributing code:

* keep changes focused and easy to review
* prefer small pull requests over large refactors
* update documentation when behavior or setup changes
* include reproduction or validation steps when fixing bugs

## Before opening a pull request

If possible:

* build the project with `cargo build --release`
* run the quick War and Peace example with `bash ./evaluation/war-and-peace/test.sh`
* run any additional benchmark or Bash test-suite checks relevant to your change

## Questions

If you are unsure whether a change fits the project, open an issue first and describe the proposed change.
